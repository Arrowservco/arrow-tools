# BidLens Architecture

## Layering

```
UI (Next.js App Router, React, Tailwind, shadcn-style components)
│   src/app/*                pages: / (analyze flow), /history, /evaluation/[id], /settings
│   src/features/*           analysis, extraction, research, results, history, settings
│
├── Deterministic core (pure TypeScript, shared client + server)
│   src/lib/money.ts         integer-cents Money utilities
│   src/lib/profile.ts       Eric Standard profile + eBay category fee table
│   src/lib/calculations/    macbid, ebayFees, shipping, profitability, maximumBid,
│                            market, velocity, risk, verdict, evaluate
│   src/lib/pipeline.ts      extraction/research → calculator inputs; assembleEvaluation;
│                            reevaluate (instant client-side recalculation)
│
├── AI layer (server only)
│   src/lib/ai/prompts/      six prompt files (extraction, identity, market, comparables,
│                            shipping, explanation)
│   src/lib/ai/schemas/      Zod schemas that every model response must satisfy
│   src/lib/ai/providers/    AIProvider interface + OpenAI / Anthropic / Gemini / Demo
│
├── API (Next.js route handlers, Node runtime)
│   /api/v1/evaluate, /evaluations/[id](/recalculate), /providers/test, /health
│   /api/v1/extract, /research, /explain   (step endpoints used by the UI)
│   src/lib/api/             auth (Bearer/same-origin), orchestrator, serverStore, shape
│
└── Storage
    src/lib/storage/db.ts        Dexie (IndexedDB) history, browser-local
    src/lib/storage/settings.ts  localStorage (settings) + sessionStorage (keys)
    src/lib/api/serverStore.ts   ephemeral file store for API-created evaluations
```

## Data flow (UI)

1. **Upload** → `POST /api/v1/extract` (image + provider config) → `MacBidScreenshotExtraction`
   (Zod-validated; every critical field is a `ConfidenceValue`).
2. **Review** → user confirms/edits fields (react-hook-form + Zod). A preliminary deterministic
   screen flags obviously unprofitable Instant Wins before any research call.
3. **Research** → `POST /api/v1/research` → identity + market comparables + shipping specs
   (bounded by `maxResearchCalls`).
4. **Assemble** → `assembleEvaluation()` runs **client-side**: market stats, shipping bands,
   scenarios, max-bid solver, velocity, risk, verdicts.
5. **Edit & recalculate** → `reevaluate()` re-runs only the deterministic core in-browser —
   instant, no AI, no network.
6. **Save** → Dexie record (result + extraction + thumbnail + actual-outcome slot).

The REST `POST /api/v1/evaluate` runs steps 1–4 server-side in one call.

## Key decisions

- **Cents everywhere.** All money is integer cents; rates are decimal fractions. `roundCents`
  rounds half away from zero at each fee step (matches receipt-style rounding).
- **Binary-search max bid.** Fees and tax depend on the bid, so the ceiling is found by a
  monotonic binary search exact to the cent, then floored to whole dollars so the recommended
  bid always still satisfies every threshold.
- **Conservative controls.** The conservative scenario (quick-sale price, conservative label,
  1.5× return reserve, accessory allowance, full promotion) gates the verdict; expected/best are
  informational.
- **AI is untrusted input.** Every model response is schema-validated (Zod) and passes through
  deterministic hard-exclusion rules (parts-only, wrong model, wrong quantity, local pickup…)
  before statistics are computed. Sold status claimed without web research is downgraded.
- **Client-recalculable results.** `EvaluationResult` carries everything needed to re-run the
  calculators, so the results screen can live-edit assumptions without touching the server.
- **Keys stay out of persistence.** Session keys live in sessionStorage only and travel only to
  this app's own API routes per request; env keys never reach the browser.
- **PWA without a framework.** Hand-rolled `sw.js` (network-first app shell) + manifest; icons
  and the demo screenshot are generated by `scripts/generate-assets.mjs` (headless Chromium).
